# Contributing

## Documentation Language

This project uses English as the single language for technical documentation.

- Write all repository documentation in English.
- Keep code comments and technical notes in English.
- Translate non-English technical content to English before submitting changes.

Applies to:

- `README.md`
- `docs/`
- technical RFCs/notes stored in the repository
- migration and incident documentation

## Documentation Consistency

If behavior changes, update documentation in the same commit/PR:

1. `docs/system-logic.md` (current logic source of truth)
2. `docs/current-status.md` (maturity, risks, priorities)
3. `docs/agent-handoff-current-main.md` (onboarding and execution context)
4. `README.md` (user-facing capability summary)

Point-in-time incident and verification reports may be kept for audit history, but should be clearly marked as historical.
